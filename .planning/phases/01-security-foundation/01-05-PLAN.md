**삽입 위치:** 기존 30일 재인증 useEffect 블록 직후 (`}, []); // eslint-disable-line` 줄 다음 빈 줄)

**추가할 코드:**
```javascript
// ── Firebase Auth 상태 동기화 (SEC-07) ───────────────────────────────────────
// onAuthStateChanged가 로그아웃을 감지하면 localStorage 세션도 제거
// Firebase 비밀번호 변경, 계정 비활성화, 토큰 만료 시 자동 로그아웃
useEffect(() => {
  const unsub = onAuthStateChanged(auth, async (fbUser) => {
    if (!fbUser && user) {
      // Firebase가 로그아웃 상태 → localStorage에만 남은 세션 제거
      setUserPersist(null);
      return;
    }
    if (fbUser && fbUser.email && user) {
      // Firebase 세션 유효 → last_login 타임스탬프 갱신
      try {
        localStorage.setItem("ryek_last_login", String(Date.now()));
      } catch {}
    }
  });
  return () => unsub();
}, [user]); // eslint-disable-line react-hooks/exhaustive-deps
```

주의사항:
1. `user` 는 MainApp의 state이므로 deps에 추가한다 (`[user]`). 단, react-hooks/exhaustive-deps eslint 규칙은 `// eslint-disable-line` 으로 억제한다 (기존 30일 체크 useEffect와 동일 패턴).
2. `setUserPersist`는 기존 코드에서 localStorage + state를 함께 업데이트하는 setter이다. null 전달 시 세션이 제거된다.
3. `auth`는 App.jsx line 2의 import에서 이미 가져오고 있다 (`import { ..., auth, ... } from "./firebase.js"`).
4. 이 useEffect는 마운트 시 한 번 실행되고, Firebase Auth 상태가 바뀔 때마다 콜백이 호출된다.
5. getIdToken 강제 갱신은 하지 않는다 — Plan 03(set-role 호출 직후)에서 이미 처리하기 때문. 이 리스너는 세션 무효화 감지에만 집중한다.
