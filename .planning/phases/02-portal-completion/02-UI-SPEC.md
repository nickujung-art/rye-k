## Spacing Scale

The project uses an ad-hoc but consistent mobile-first spacing pattern. The following values are in use and MUST be observed:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, day chip inner padding |
| sm | 8px | Compact gaps, badge padding, chip margins |
| md | 16px | Default section padding, card padding |
| lg | 24px | Section breaks, login card padding |
| xl | 32px | Major section separation |
| 2xl | 48px | Empty state top padding |
| 3xl | 64px | Page-level spacing (rarely used) |

**Exceptions for this phase:**
- Bottom nav safe-area clearance: `calc(var(--nav-h) + var(--safe-b) + 16px)` вҖ” used as `padding-bottom` on scrollable content
- Touch targets for nav items, buttons: minimum 44px height (already enforced via `--nav-h` and `.btn` / `.att-btn`)
- Session expiry banner: 16px vertical padding, full-width sticky (see Banner spec below)

## Interaction Contracts

### Session Expiry Flow
1. On mount of `PublicParentView`, read `localStorage["ryekPortal"]` вҶ’ extract `loginAt`.
2. If `Date.now() - loginAt > 30 * 24 * 60 * 60 * 1000`: clear storage, `setLoggedIn(false)` вҶ’ login screen.
3. If `Date.now() - loginAt > 27 * 24 * 60 * 60 * 1000`: set `showExpiryBanner = true`.
4. "30мқј м—°мһҘ" tap: `loginAt = Date.now()`, write back to localStorage, `setShowExpiryBanner(false)`.
5. "лЎңк·ём•„мӣғ" tap: clear localStorage, `setLoggedIn(false)`. Immediate.

### Schedule Widget Rendering
1. Read `student.lessons[]` вҖ” iterate all lessons.
2. For each lesson, find minimum next-occurrence day: compare `DAYS` index against `TODAY_DAY`.
3. Next occurrence = nearest upcoming `l.schedule[].day` + `l.schedule[].time`.
4. Resolve teacher name: `teachers.find(t => t.id === l.teacherId)?.name` вҶ’ fallback to "к°•мӮ¬ лҜёл°°м •".
5. If no lessons: render nothing (no widget container).
6. If multiple lessons: render one next-lesson card per lesson, stacked, 8px gap.

### Tab Swipe
Already implemented via `onTouchStart` / `onTouchEnd`. No changes needed.

### Text Large Toggle
Already implemented via `textLarge` state and `.text-large` class on portal root. Portal body scales per existing CSS. No changes.
