# Plan 07-01: guardianName 필드 + students_cache 개선 + sync 타이밍 수정

## Objective

학생 데이터에 보호자 이름(guardianName)을 추가하고, students_cache를 항상 최신 상태로 유지한다.
이 플랜은 Wave 1로, 07-02(알고리즘)와 07-03(UI)의 전제 조건이다.

**배경**: Tasker가 입금 알림을 webhook으로 보낼 때 `students_cache`가 비어있으면 무조건 unmatched.
현재 sync는 payments 탭을 열 때만 실행되고, 24h TTL이어서 캐시가 자주 만료된다.
또한 보호자가 자기 이름으로 입금하는 경우가 많으나 DB에 guardianName 필드 자체가 없다.

  - line ~235~244: useState 목록 (unmatchedPayments, paymentLog 등)

<action>
1. payments 탭 진입 시 sync body의 students.map 수정 (line 574~582):
   ```js
   // 기존

     id: s.id,
     name: s.name,
     status: s.status || "active",

   }))
   // 변경 후
   students.map(s => ({

     status: s.status || "active",
     isInstitution: s.isInstitution || false,
     guardianName: s.guardianName || "",
     monthlyFee: s.monthlyFee || 0,
   }))
   ```

2. 앱 로드 시 추가 sync — students Firestore 리스너 완료 콜백 내부에 best-effort sync 추가.
   Firestore onSnapshot 내에서 students setStudents() 호출 후, 또는 별도 useEffect로:
   ```js
   // MainApp 내 students 로드 후 실행되는 useEffect 추가
   // (students.length > 0이고 user가 canManageAll인 경우에만 실행)
   // 이미 sync 실행 중이면 skip — 중복 실행 방지용 ref 사용
   const syncedOnMount = useRef(false);
   useEffect(() => {
     if (!user || !canManageAll(user.role) || students.length === 0) return;
     if (syncedOnMount.current) return;
     syncedOnMount.current = true;
     // best-effort, silent
     (async () => {
       try {
         const currentUser = auth.currentUser;
         if (!currentUser) return;
         const token = await currentUser.getIdToken();
         await fetch("/api/payments/sync-students", {
           method: "POST",
           headers: { "Content-Type": "application/json", Authorization: `Bearer ${token}` },
           body: JSON.stringify({
             role: user.role,
             students: students.map(s => ({
               id: s.id,
               name: s.name,
               status: s.status || "active",
               isInstitution: s.isInstitution || false,
               guardianName: s.guardianName || "",
               monthlyFee: s.monthlyFee || 0,
             })),
           }),
         });
       } catch { /* silent */ }
     })();
   }, [user, students.length > 0]);
   ```
   의존성: `user?.role`, `students.length` (0→양수 전환 시 1회만 실행).
   `syncedOnMount` ref로 중복 방지. payments 탭 기존 sync와 무관하게 독립 동작.
</action>

<acceptance_criteria>

- App.jsx에 `syncedOnMount` useRef 추가
- 앱 마운트 후 students 로드 시 1회 best-effort sync 실행 (canManageAll 조건)
