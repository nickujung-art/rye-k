## Current Testing Approach

**There is no automated test suite.** The project has zero test files, no test runner, no coverage tooling.

```json
// package.json scripts — no "test" script
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}
```

Validation relies entirely on:
1. `npm run build` — Vite/Rollup compilation succeeds
2. Manual browser testing against `npm run dev` or `npm run preview`

---

## What `npm run build` Catches

| Catches | Does NOT Catch |
|---------|----------------|
| Syntax errors in JSX/JS | Logic bugs |
| Missing named exports (import errors) | Data loss conditions |
| Dead import paths | Auth/role edge cases |
| Circular dependency crashes | Firebase read/write errors |
| Rollup bundle errors | Runtime permission violations |
| Vite plugin errors | UI state management bugs |

No TypeScript — the codebase is plain `.js` / `.jsx`, so there is no type checking at build time either.

## Manual Testing Areas

### Critical paths requiring browser verification
- **Student CRUD**: Add, edit, delete, bulk-fee update via Firestore transaction functions
- **Institution CRUD + virtual member expansion**: Institution classes appearing correctly in attendance/schedule
- **Attendance recording**: Per-student status, bulk mark, late/excused flows
- **Auth flows**: Admin login, teacher login, anonymous visitor, portal (student code)
- **Role-based visibility**: Teacher sees only own students; fees hidden from teacher role
- **AI features**: Lesson note generation, monthly report, churn widget
- **Update popup**: First-login version check triggers popup, subsequent logins skip it

### Firebase-specific manual tests
- Real-time sync across two browser tabs
- Offline behavior (Firestore cache)
- Auth session persistence across page reload
- Transaction retry on concurrent edit

## Known Coverage Gaps

### No unit tests for pure utility functions (`src/utils.js`)
The following functions have zero test coverage and contain business-critical logic:
- `calcAge(birthDate)` — student age calculation
- `expandInstitutionsToMembers(institutions)` — virtual member generation
- `allLessonDays(student)` — schedule aggregation
- `generateStudentCode()` — unique code generation
- `getBirthPassword(birthDate)` / `getPhoneInitialPassword(phone)` — initial password derivation
- `fmtMoney(amount)` — financial display formatting
- `formatLessonNoteSummary(note)` — note display logic

### No integration tests for Firestore transaction functions
- `addStudentDoc`, `updateStudentDoc`, `deleteStudentDoc`, `batchStudentDocs`
- These are the most critical functions in the codebase — a regression here causes data loss

### No E2E tests for critical user flows
- Student registration → attendance → payment flow
- Teacher login → view assigned students only (role isolation)
- Institution class → virtual member injection into attendance view

### No visual regression tests
- CSS is injected as inline `<style>` tags from JS string constants
- No tooling to catch CSS regressions

## Recommended Testing Stack

### Priority 1: Unit tests for utils.js (immediate value, low effort)

```bash
npm install -D vitest @vitest/ui jsdom
```

```js
// vite.config.js addition
test: {
  environment: 'jsdom',
  globals: true,
}
```

```js
// src/utils.test.js
import { describe, it, expect } from 'vitest'
import { calcAge, expandInstitutionsToMembers, fmtMoney } from './utils'

describe('calcAge', () => {
  it('returns correct age from birthdate string', () => {
    expect(calcAge('2000-01-01')).toBe(25) // adjust for test date
  })
})
```

Target: 80% coverage of `src/utils.js` functions.

### Priority 2: Firestore transaction function tests (medium effort, critical)

Use Firebase Emulator Suite:
```bash
npm install -D @firebase/rules-unit-testing firebase-admin
```

Mock Firestore locally to test that:
- `addStudentDoc` appends without overwriting existing records
- `deleteStudentDoc` removes only the target ID
- Concurrent `updateStudentDoc` calls resolve correctly via transaction retry

### Priority 3: E2E with Playwright (higher effort, high value)

```bash
npm install -D @playwright/test
npx playwright install
```

Critical flows to cover:
- Login → view dashboard → mark attendance
- Add student → verify appears in list
- Teacher login → verify cannot see other teacher's students
- Institution class → verify virtual member in attendance view
