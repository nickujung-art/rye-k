### Task 2: amount_match 추천 학생 자동 선택

<read_first>
- src/components/payment/PaymentsView.jsx
  - `UnmatchedPaymentsTab` 컴포넌트 (line ~1289)
  - `selectedStudentId` state (line ~1294): `{}` 객체, key = u.id
  - `pending.map(u => ...)` 내부 학생 선택 `<select>` 블록
  - `handleMatch` 함수 (line ~1317): `selectedStudentId[unmatched.id]`로 학생 찾음
</read_first>

<action>
1. `useEffect` 추가 — pending 목록이 변경될 때 `suggestedStudentId` 있는 항목을 자동으로 `selectedStudentId`에 설정:

```jsx
// UnmatchedPaymentsTab 내부
useEffect(() => {
  const autoSelections = {};
  pending.forEach(u => {
    if (u.suggestedStudentId && !selectedStudentId[u.id]) {
      autoSelections[u.id] = u.suggestedStudentId;
    }
  });
  if (Object.keys(autoSelections).length > 0) {
    setSelectedStudentId(prev => ({ ...prev, ...autoSelections }));
  }
}, [pending.length]);
```

2. `<select>` 위에 추천 배지 표시:
```jsx
{u.confidence === "amount_match" && u.suggestedStudentId && (
  <div style={{
    fontSize: 11,
    color: "var(--blue, #1976d2)",
    marginBottom: 4,
    fontWeight: 600,
  }}>
    💡 금액 기반 추천: {students.find(s => s.id === u.suggestedStudentId)?.name || "알 수 없음"}
  </div>
)}
```
위치: rawText 표시 div 아래, `<select>` 위.

3. `<select>` 표시 레이블 추가 — amount_match일 때 "(추천)" suffix:
   현재 select는 options에 학생 이름을 렌더링함. 추천 학생 option에 "(추천)" 추가:
   ```jsx
   <option key={s.id} value={s.id}>
     {s.name}{u.suggestedStudentId === s.id && u.confidence === "amount_match" ? " (추천)" : ""}
   </option>
   ```
</action>

<acceptance_criteria>
- `suggestedStudentId` 있는 pending 항목이 마운트 시 자동으로 `selectedStudentId`에 설정됨
- `confidence === "amount_match"` + `suggestedStudentId` 있을 때 "💡 금액 기반 추천: [이름]" 표시
- 추천 학생 option에 " (추천)" suffix 표시
- useEffect 의존성: `[pending.length]` (불필요한 재실행 방지)
- 기존 수동 선택 동작 변경 없음
- `window.confirm` / `window.alert` 없음
</acceptance_criteria>
