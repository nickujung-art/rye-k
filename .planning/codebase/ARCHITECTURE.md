## System Overview

RYE-K is a **Single Page Application (SPA)** serving as an all-in-one management system for a K-Culture (Korean traditional music) education center.

```
┌─────────────────────────────────────────────────────────┐
│                    Client (Browser/PWA)                  │
│  React 18 SPA — ~880KB bundle (~220KB gzip)             │
│  Vite 5 build → dist/ → Cloudflare Pages               │
└──────────────────────┬──────────────────────────────────┘
                       │ Firestore real-time onSnapshot
                       │ Firebase Auth (anonymous + email)
┌──────────────────────▼──────────────────────────────────┐
│                  Firebase (Google)                       │
│  Firestore: single "appData" collection                  │
│  Auth: anonymous sign-in + email/password               │
└──────────────────────┬──────────────────────────────────┘
                       │ HTTPS fetch from browser
┌──────────────────────▼──────────────────────────────────┐
│           Cloudflare Workers (functions/api/ai/)         │
│  AI endpoints: lesson-note, monthly-report, churn,      │
│  query, reply-suggest, payment-tone, practice-guide,    │
│  punctuate                                               │
│  Middleware: auth guard + rate limiter (KV)             │
│  LLM backend: Anthropic Claude (via anthropic.js)       │
└─────────────────────────────────────────────────────────┘
```

## Data Flow

### Real-time Listener Pattern

`App.jsx` establishes all Firestore `onSnapshot` listeners on login and tears them down on logout.

```
Firebase Auth state change
  └─ onAuthStateChanged()
       └─ firebaseSignIn() / firebaseSignInAnon()
            └─ onSnapshot(doc(db, "appData", key))  [per collection]
                 └─ setState(parsedData)  [React state]
                      └─ props / context passed to child components
```

All application data lives in React state in `App.jsx` and is passed down as props. There is no global state library (no Redux, no Zustand).

### Write Path — Critical Safety Rule

**NEVER**: `sSet("rye-students", allStudentsArray)` — this overwrites all data atomically and is permanently banned after a 77-student data loss incident.

**ALWAYS**: Per-operation Firestore transaction functions:
```
addStudentDoc(student)        → runTransaction: read current array → append → write
updateStudentDoc(student)     → runTransaction: read current array → replace by id → write
deleteStudentDoc(studentId)   → runTransaction: read current array → filter out → write
batchStudentDocs(updates[])   → runTransaction: read current array → multi-update → write
```

This pattern reads from Firestore (not React state) before every mutation, preventing race conditions where filtered views would cause unseen records to be dropped.

The same per-op pattern should be applied to all collections but currently is only enforced for students.

## Auth Architecture

### Auth Modes
| Mode | Who | How |
|------|-----|-----|
| Anonymous | New visitors | `firebaseSignInAnon()` — silent auto sign-in |
| Email/Password | Staff (teachers, admin, manager) | `firebaseSignIn(username, password)` |
| Portal | Students/parents | `studentCode` lookup, no Firebase Auth |

### Role System
| Role | Firebase Auth | Access Level |
|------|--------------|--------------|
| `admin` | Email auth | Full access, all data, all teachers |
| `manager` | Email auth | Full access, all data, all teachers |
| `teacher` | Email auth | Own students only, no fees, no peer contacts |

Role is stored as `role` field on the teacher record in `rye-teachers`, not in Firebase Auth claims.

### Auth State Storage
- Firebase Auth: handles token refresh
- `localStorage["rye-session"]`: stores teacher profile (username, role, teacherId)
- **Known issue**: Local session never expires; Firebase Auth and local session can desynchronize

## AI Integration Architecture

All AI features route through Cloudflare Workers in `functions/api/ai/`.

```
Browser → src/aiClient.js
  └─ fetch("/api/ai/<endpoint>", { headers: { Authorization: "Bearer <firebase-id-token>" }})
       └─ functions/api/ai/_middleware.js
            ├─ _utils/auth.js      → verify Firebase ID token
            ├─ _utils/ratelimit.js → KV-based rate limiting (silently skipped if KV missing)
            └─ <endpoint>.js
                 └─ _utils/anthropic.js → callAnthropic() → Anthropic Claude API
```

### AI Endpoints
| Endpoint | Purpose |
|----------|---------|
| `lesson-note` | Generate AI lesson note from voice/text input |
| `monthly-report` | Generate monthly student progress report |
| `churn` | Predict student churn risk |
| `query` | Natural language query over student data |
| `reply-suggest` | Suggest reply to student/parent message |
| `payment-tone` | Generate payment reminder message |
| `practice-guide` | Generate practice guide for student |
| `punctuate` | Add punctuation to voice-transcribed text |

**Known issue**: `_utils/anthropic.js` is named for Anthropic but implementation calls Claude via Anthropic SDK correctly — the confusion is the file was originally Gemini-based and has been migrated.

## Role-Based Access Control

| Feature | admin | manager | teacher |
|---------|-------|---------|---------|
| All students | ✓ | ✓ | Own only |
| Student fees | ✓ | ✓ | Hidden |
| Institution management | ✓ | ✓ | View own only |
| Teacher contacts | ✓ | ✓ | Hidden |
| Admin tools | ✓ | ✗ | ✗ |
| AI settings | ✓ | ✗ | ✗ |
| Trash / Activity | ✓ | ✗ | ✗ |

`canManageAll(teacher)` utility: returns `true` for admin and manager roles.

## Virtual Member Pattern

Institutions (B2B clients — schools, government, companies) are stored as `Institution` documents with nested `classes[]`. For views that need to treat institution classes as "students" (attendance, schedule, payment), the helper `expandInstitutionsToMembers()` in `src/utils.js` converts each institution class into a virtual member object at runtime.

```
rye-institutions[] → expandInstitutionsToMembers() → virtualMember[]
  { id: "inst_<instId>_<classId>", isInstitution: true, name: "기관명 - 반명", ... }
```

Virtual members are **never saved to Firestore** — they are runtime-only and regenerated from institution data on each render cycle.

**Critical rule**: `isInstitution: true` members must be excluded from `StudentsView` and `Dashboard.students` counts. They are only injected into Attendance, Schedule, and Payment views.
