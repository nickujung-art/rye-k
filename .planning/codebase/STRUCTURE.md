## Firestore Collections Map

All data stored under single Firestore collection `appData`, with document keys:

| Key | Type | Description |
|-----|------|-------------|
| `rye-teachers` | `Teacher[]` | Staff: teachers, managers, admin |
| `rye-students` | `Student[]` | Active/paused/withdrawn members |
| `rye-attendance` | `Attendance[]` | Per-lesson attendance records |
| `rye-payments` | `Payment[]` | Fee payment records |
| `rye-notices` | `Notice[]` | Announcements |
| `rye-categories` | `{ [catName]: string[] }` | Instrument category groupings |
| `rye-fee-presets` | `object` | Tuition + rental fee presets |
| `rye-schedule-overrides` | `ScheduleOverride[]` | Temporary schedule changes |
| `rye-activity` | `ActivityLog[]` | Audit log (max 200, then truncated) |
| `rye-pending` | `Pending[]` | Teacher self-registration requests |
| `rye-trash` | `TrashItem[]` | Soft-deleted records |
| `rye-student-notices` | `StudentNotice[]` | Per-student notice delivery |
| `rye-institutions` | `Institution[]` | B2B organization contracts (v12.1+) |

**Scalability note**: All records of each type are stored as one array in one Firestore document. At ~1MB per document this will eventually fail for high-volume collections (attendance, payments, activity).

## Key Data Structures

### Student
```js
{
  id: string,
  name: string,
  birthDate: string,         // "YYYY-MM-DD"
  startDate: string,
  phone: string,
  guardianPhone: string,
  teacherId: string,
  lessons: [
    {
      instrument: string,
      teacherId: string,
      schedule: [{ day: string, time: string }]
    }
  ],
  photo: string,             // base64 or URL
  notes: string,
  monthlyFee: number,        // currently 0 for all records
  status: "active" | "paused" | "withdrawn",
  studentCode: string,       // "RK" + 4 chars, e.g. "RKAB12"
  createdAt: number          // Unix ms
}
```

### Institution
```js
{
  id: string,
  name: string,
  type: "school" | "center" | "company" | "government" | "other",
  address: string,
  contactName: string,
  contactPhone: string,
  contactEmail: string,
  bizNumber: string,
  teacherId: string,
  classes: [
    {
      id: string,
      name: string,
      instrument: string,
      teacherId: string,
      schedule: [{ day: string, time: string }],
      participantCount: number,
      monthlyFee: number,
      notes: string
    }
  ],
  contractStart: string,
  contractEnd: string,
  status: "active" | "paused" | "expired",
  notes: string,
  photo: string,
  createdAt: number
}
```

### Virtual Member (runtime only вҖ” NOT saved to Firestore)
```js
{
  id: "inst_<instId>_<classId>",
  isInstitution: true,
  name: "<instName> - <className>",
  // ... other student-compatible fields for attendance/payment views
}
```

### Attendance Record
```js
{
  id: string,
  studentId: string,
  teacherId: string,
  date: string,              // "YYYY-MM-DD"
  status: "present" | "absent" | "late" | "excused",
  lessonNote: string,
  note: string,
  comments: [
    {
      id: string,
      text: string,
      authorType: "teacher" | "manager" | "admin" | "student",
      authorName: string,
      authorId: string,
      createdAt: number,
      deletedAt: number | null,
      deletedBy: string | null
    }
  ],
  participantCount: number,  // for institution virtual members
  createdAt: number,
  updatedAt: number
}
```
